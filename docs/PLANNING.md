# Planning Guide

## Claude Code와 함께 플래닝하기

이 문서는 Claude Code를 활용하여 효과적으로 프로젝트를 계획하고 실행하는 방법을 안내합니다.

---

## 1. 새 기능 계획하기

새 기능을 개발하기 전에 Claude에게 다음과 같이 요청하세요:

```
[기능명]을 개발하려고 해. PRD에 기능을 추가하고 계획을 세워줘.
```

Claude는 다음을 수행합니다:
1. PRD.md에 기능 요구사항 추가
2. FEATURES.md에 상세 명세 작성
3. 구현 계획 제안

---

## 2. 스프린트 계획

스프린트를 시작할 때:

```
이번 스프린트에서 [기능 목록]을 개발하려고 해. 플래닝 해줘.
```

Claude는:
1. FEATURES.md의 상태를 업데이트
2. 작업 순서 및 의존성 분석
3. 세부 태스크 분해

---

## 3. 일일 작업 시작

```
오늘 F-001 기능을 개발할 거야. 시작해줘.
```

Claude는:
1. 해당 기능의 상세 명세 확인
2. 관련 코드 분석
3. 구현 시작 및 진행 상황 추적

---

## 4. 진행 상황 업데이트

```
F-001 기능 개발이 완료됐어. PRD 업데이트 해줘.
```

Claude는:
1. PRD.md의 상태를 🟢 Done으로 변경
2. FEATURES.md의 완료 섹션으로 이동
3. 변경 이력 업데이트

---

## 5. 문서 리뷰

```
현재 PRD 상태를 요약해줘.
```

Claude는:
1. 전체 진행 상황 요약
2. 완료/진행중/대기 기능 목록
3. 다음 작업 추천

---

## Quick Commands

| 명령어 | 설명 |
|--------|------|
| `PRD에 [기능] 추가해줘` | 새 기능 요구사항 추가 |
| `F-XXX 상태를 [상태]로 변경해줘` | 기능 상태 업데이트 |
| `현재 진행 상황 보여줘` | 프로젝트 상태 요약 |
| `다음 작업 추천해줘` | 우선순위 기반 작업 추천 |
| `[기능] 구현 계획 세워줘` | 상세 구현 계획 생성 |

## 파일 구조

```
docs/
├── PRD.md           # 제품 요구사항 문서 (메인)
├── FEATURES.md      # 기능 상세 추적
├── PLANNING.md      # 플래닝 가이드 (현재 파일)
└── CHANGELOG.md     # 버전별 변경 이력 (선택)
```

---

## 상태 흐름

```
🔴 Not Started
      ↓
🟡 In Progress
      ↓
🔵 Testing / Review
      ↓
🟢 Done

(언제든 ⚫ Blocked 상태로 전환 가능)
```

---

## Tips

1. **PRD를 항상 최신 상태로 유지하세요**
   - 기능 개발 전후로 문서 업데이트

2. **작은 단위로 계획하세요**
   - 큰 기능은 여러 개의 작은 태스크로 분해

3. **블로커를 즉시 기록하세요**
   - 진행이 막히면 바로 문서에 기록

4. **정기적으로 리뷰하세요**
   - 매일 또는 매주 진행 상황 확인
